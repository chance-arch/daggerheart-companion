## Design & architecture decisions (durable record)

1. **Splash dismissal is user-gated, with a keyboard path.** Auto-close was the only non-pointer
   way past the splash; removing it without Enter/Space/Escape would have made the app
   mouse-gated. Accessibility kept deliberately (hard-line item).
2. **The splash key listener detaches on close** — it `preventDefault()`s Space/Enter/Escape on
   `document`, so leaving it attached would swallow spacebar in the notes textarea for the whole
   session. Load-bearing, not hygiene (verified by a dedicated check).
3. **No skip-splash preference / configurable timeout** — deliberate YAGNI. Splash shows every
   launch, holds until tapped, done. Hint fade-in delay (2.2 s) kept so the loading gag lands
   before the hint gives it away (ponytail-audited in-session; Chance can cut with one line).
4. **`build_data_index.py` is a refresher, not a generator.** It recomputes only the "At a
   glance" table + Generated date from the JSONs and leaves all prose alone. A full generator
   would have to embed the hand-written schema/cross-reference prose — over-engineering for a
   file that changes rarely.
5. **The `.xlsx` twin mirrors the whole JSON, including `meta.priceModel`** — the twin's job is
   review, and reviewing rarity without the price tables it keys into is half a picture.
6. **`meta.sourceTier` is now standardized across all 9 datasets** — the precedence policy
   (`SOURCES_precedence.md`) is only mechanical if every dataset carries the tag.
